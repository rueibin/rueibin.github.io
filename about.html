接觸過的ERP系統

(1)上華ERP

套裝系統，Cient Server ERP。

第一次接觸ERP系統。當時，覺得系統很大，很難，很陌生，不會操作。心理想怎麼學丫（苦笑...），當時有請教主管與同事，結果，同事只會操作局部的，問原理與流程，就不會。 我得到一個結論，他們只會局部的，如果要彙整全部的，就不會。畢竟，要一個非資訊員彙整全部ERP流程有點難，但是還是有機會的，例如：對公司業務流程熟悉的就可以。

所以，當時的我對上華ERP的操作，只限於清ERP資料庫的暫存而已，操作就放棄了。

(2)科展ERP

客制系統，Cient Server ERP，有基礎模組，在基礎模組之上加上客制化的元素，不提供原始碼。

當時上華ERP已經不滿足公司業務的需求，所以要更換ERP系統。 找三家ERP廠商，最後科展ERP得標。 決定廠商與簽約之後，就開始展開系統的導入，這次要導入ERP有十幾個模組（年代久遠，只記得十幾個，到底實際多少個忘了）。 導入ERP系統，公司組織了4個人的Team，二個高階主管，一個倉庫組長，一個資訊人員。 資訊人員的我，負責整個系統的導入時間規劃，資料收集，教育訓練與，每次meeting的會議記錄與對外窗口。 整個系統的導入，分為好幾個階段：

初期階段：討論各階段上線的時間表與系統所需要的報表資料。 對於編碼原則與報表資料我印象深刻，當時的我從上華ERP列印出各部門所需要的報表，對每張報表的每個欄位，依依請教每位同事，才把報表資料整理完成，而不是請各部門列印交上來。 編碼原則，我是網路參考了好幾十個編碼範例，然後與各部門討論了好幾次才製作出來的編碼原則手冊（我覺得編碼原則很難，太多與太少都不好）

第一階段上線：上線的內容是跟成本比較沒有關系。上線的模組是基本資料系統，人事系統與模具系統。 第一階段上線前還要將基本資料匯入到系統，如廠商、客戶、產品、BOM表...等資料。還有重點工作站要配置電腦與條碼刷。 會要求ERP廠商提供操作手冊，但是，我還是會寫屬於公司的操作手冊。 這個階段上線的系統比較單純，ERP廠商可以不用駐點。

第二階段上線：這個階段很重要，幾乎全部的系統要上了，上線系統大概提一下（年代久遠，有點忘了），有訂單，採購、出貨、庫存、會計等系統。 ERP廠商要駐點，主要是解決剛上線時的bug能即時修正。我還要記錄系統bug回報給系統廠商。

最後階段上線與維護階段：這個階段，把之前沒有上的系統全部上線，還要結成本（我沒有負責成本這一塊）。 這個階段，我還是一樣忙碌，忙著寫操作手冊，忙著當客服回答與解決同事ERP操作問題，忙著跟廠商回報問題。

結論：

這套系統大概花了兩年導入，前半年都是需求討論與資料蒐集，後面才是系統真正導入。 感謝能夠參與ERP的導入，導入的工作，我都是親力親為的，這也讓我體會了負責專案的不為人知辛苦的一面與ERP這個領域的入門知識，唯讀缺了會計這方面的知識。 但，這只了解科展ERP操作而已，並不代表你了解ERP，還是需要的歷練才會成長。

(3)天心ERP

套裝系統，Cient Server ERP，有提供源始碼的系統，使用VB開發。

一聽到使用VB開發，VB有點老，頭部就有好幾隻烏鴉飛過。不過，學程式不難，只要掌握基本的操作與資料庫讀寫就可以開始寫了。

主要負責HR系統（兩個廠的HR系統，台灣總廠與大陸廠）與支援其他系統維護，因為有維護過HR系統，所以被分派到負責HR系統。 第一個處理HR系統的問題就是薪資計算，當時，來自大陸廠的一位人資提出，某位員工薪資多算了。 那我首先查看code，了解薪資計算的規則，debug時發現某個條件沒有滿足，導致薪資多算了。 修改規則（修改人事規則前，要先問主管），薪資就計算正常。

(4)客制內部系統

客制系統，Cient Server ERP，有提供源始碼的系統，使用Visual FoxPro開發。

使用Visual FoxPro，又是一個古老的語言，但，不影響開發。 有很多獨立的系統，靠著資料庫關連起來的，有會計、預算、固資、停車場收費等系統。 只有會計系統改的比多。

(5)鼎新Workflow

套裝系統，Cient Server ERP。

有之前接觸過ERP的經驗，容易上手。 修改頻繁的工作是報表，因為可以建立客制化的報表，所以常會新增或修改報表。

(6)貫智進銷存

套裝系統，Cient Server ERP。

小系統，有採購、銷貨、應收、應付等功能，容易上手。

(7)WEB進銷存

客制系統，Browser Server，提供原始碼。

公司的需求開發內部系統。 第一次開發進銷存，採用SSM，資料庫使用MySQL，前端+後端。

功能，如下：

  1. 公司管理：公司設定
  2. 供應商管理：廠供查詢，供應商設定，付款條件
  3. 客戶管理：客戶設定
  4. 商品管理：商品單位設定，商品分類設定，商品設定
  5. 權限管理：資源設定，群組設定，帳號設定
  6. 進貨管理：採購單，驗收單，進貨單，退貨單，進貨折讓單
  7. 銷貨管理：報價單，銷貨單，撿貨單，銷退單
  8. 庫存管理：庫存調整單，即時庫存查詢，即時寄庫庫存查詢，報廢單，領用單
  9. 進項發票管理：進貨發票設定，進貨退出證明單，進貨折讓證明單，進貨發票勾稽作業，進貨發票反勾稽作業
  10. 應付管理：應付帳款
  11. 銷項發票管理：銷貨發票開立，銷貨發票設定
  12. 應收管理：應收帳款

現在才發現一個人開發進銷存，即使是小系統，還是很困難，因為要考慮的細項一堆， 而且不同領域的知識也非常多，都要靠自己去找資料，依依排除與解決。 目前，系統開發中
